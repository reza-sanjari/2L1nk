Self-hosting 2L1nk

2L1nk is a self-hosted, end-to-end encrypted chat server. All encryption happens in the browser — the server never sees your messages or private keys.

1 — Download
1

Grab the latest binary for your platform from the Releases page.

View Releases

Available builds: linux/amd64, windows/amd64, darwin/amd64

2

Make the binary executable (Linux / macOS):

chmod +x 2L1nk
2 — Run
1

Simply run the binary. On first launch it reads configuration from environment variables and opens the interactive TUI.

# TUI mode — interactive control panel (default)
./2L1nk

# Headless server — no TUI, just run the server (for VPS / systemd)
./2L1nk --server
In TUI mode you start and stop the server from inside the control panel — the binary itself stays open as the management UI. Use --server only when running unattended (e.g. as a systemd service).
2

On first run a gate token is automatically generated. Share it with users so they can register — think of it as an invite code. You can view, rotate, and limit it at any time from the TUI's Gate Key screen.

3 — Configuration

All configuration is via environment variables. Only set what you need — sensible defaults are used for everything else.

PORT          # HTTP listen port          (default: 8080)
DB_PATH       # SQLite database file path (default: 2L1nk.db)
STUN_URLS     # Comma-separated STUN server URLs (optional)
TURN_URLS     # Comma-separated TURN server URLs (optional)
TURN_USERNAME # TURN credential username   (optional)
TURN_PASSWORD # TURN credential password   (optional)
STUN/TURN are only needed for voice calls across strict NATs. For most setups you can leave them unset.
4 — TUI Control Panel

Running ./2L1nk without flags opens the interactive control panel. Navigate with ↑↓ or j/k, confirm with Enter, go back with Esc or q.

Item What it does
Run Server Starts the chat server as a detached background process. Tunnels marked Auto Start are launched automatically alongside it.
Stop Server Kills the background server process. If Temp Server mode is on, the database and log are securely wiped on stop.
Gate Key → Manage the invite token users need to register.
  • Toggle Unlimited or set a max-uses cap — the token expires after that many registrations.
  • Copy Key — copies the current token to the clipboard.
  • Generate New Random Key — rotates to a fresh random token immediately.
  • Set Custom Key — enter any string as the token.
  • View Past Keys — history of all previously used tokens.
View Logs Streams the live server log file in a scrollable view. Greyed out when No Logs is enabled in Options.
Outbound Tunnels → Manage tunnels that expose your server to the internet without port-forwarding. Built-in presets for Cloudflare, localhost.run, Serveo, Pinggy, and srv.us. Each tunnel can be started, stopped, and set to auto-start with the server. The detected public URL is shown live once the tunnel is running.
Links → Quick-access panel showing your server's local network URL, global (internet-facing) IP, and the public URLs of any running tunnels. Select an entry to view a QR code you can scan to open the app.
Reset Database Stops the server if running, deletes the database file, and re-runs migrations to create a fresh empty database. All accounts, rooms, and messages are permanently lost.
Options → Toggle persistent server settings:
  • No Logs — disables log file creation entirely.
  • Temp Server — securely wipes the database and log file when the server is stopped. Useful for ephemeral / one-session deployments.
☢ Nuke Irreversibly wipes everything: database, logs, PID file, options, tunnel config, and all tunnel log files. Files are overwritten with random bytes before deletion. The TUI exits immediately after. Use only if you want to completely remove all traces of the server.
5 — Reverse Proxy (recommended)

For production, put 2L1nk behind a reverse proxy such as nginx or Caddy to handle TLS. WebSocket connections use the /api/ws path — make sure your proxy forwards upgrade headers.

# Caddy — handles TLS automatically
your-domain.com {
    reverse_proxy localhost:8080
}

# nginx
server {
    listen 443 ssl;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
    }
}
HTTPS is required for the Web Crypto APIs used by E2E encryption — the app will not work over plain HTTP in modern browsers.
6 — Keep it Running (Linux / systemd)
1

Create a systemd service to start 2L1nk automatically on boot and restart it on crash. Use --server so it runs headless with no TUI.

# /etc/systemd/system/2l1nk.service
[Unit]
Description=2L1nk encrypted chat server
After=network.target

[Service]
Type=simple
User=youruser
WorkingDirectory=/opt/2l1nk
ExecStart=/opt/2l1nk/2L1nk --server
Restart=on-failure
RestartSec=5
Environment=PORT=8080
Environment=DB_PATH=/opt/2l1nk/2l1nk.db

[Install]
WantedBy=multi-user.target
2

Enable and start the service:

sudo systemctl daemon-reload
sudo systemctl enable 2l1nk
sudo systemctl start  2l1nk
sudo systemctl status 2l1nk   # verify it's running
3

View live logs:

sudo journalctl -u 2l1nk -f
When running under systemd you can still open the TUI on the same machine to manage the gate key or check tunnels — just run ./2L1nk without --server alongside the running service.
7 — Security Notes

What the server stores: usernames, public keys (Ed25519 + X25519), and encrypted message ciphertexts. The server has no access to plaintext messages or private keys — these never leave the user's device.

Ephemeral users: users who register with temporary mode are stored in the database for the duration of their session (so they can join rooms) but their messages are never persisted and their account is removed on disconnect.

Key rotation: the room encryption key rotates automatically every time a user joins or leaves. Old keys are discarded; new members cannot read previous messages.

Gate token: keep it private — anyone with it can create an account on your server. Use the max-uses limit to issue single-use or limited-use invites.

Keep regular backups of your database file (DB_PATH). If it's lost, all user accounts and room memberships are gone — message history is unrecoverable.
